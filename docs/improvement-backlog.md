# 体验与发布改进 Backlog

**更新日期**：2026-04-19  
**排序原则**：优先减少真实使用摩擦，其次优化视觉一致性，最后再考虑增强功能。

## P0：优先处理

### 修复无标题摄影作品的页面标题

问题：

- 线上摄影详情页出现 `undefined | Joey's Notes` 浏览器标题。
- 这会影响专业感、分享标题和 SEO。

建议：

- 摄影详情页标题 fallback 使用日期或“摄影作品”。
- `BaseLayout` 的 `title` 入参不应收到 `undefined` 拼接结果。
- 对无 `title` 的摄影内容增加专门验收。

验收：

- 无标题摄影作品的浏览器标题不出现 `undefined`。
- 页面内也不强制显示“未命名作品”。
- 浏览器 title、Open Graph title、Twitter title、页面可见标题、sitemap 或构建产物中均不出现 `undefined`、`null`、空字符串拼接。
- 无标题摄影作品仍能从列表、胶片条和详情页正常浏览。

### 摄影列表页信息密度继续收口并移除重复日期

问题：

- 摄影列表页承担作品入口和归档浏览双重任务。
- 日期、相机、标题、按钮如果同时出现，容易削弱图片优先级。
- 线上可访问树显示归档卡片日期重复出现，说明 DOM 层也存在重复信息。

建议：

- 归档卡片默认只显示最必要 meta。
- 详细 EXIF 留给摄影详情页。
- featured 区继续避免明显卡片感，让图片本身做主视觉。
- 日期只保留一处，避免视觉和辅助技术重复读取。

验收：

- 首屏第一眼看到的是照片，而不是控件或说明。
- 手机端归档卡片高度稳定，没有明显断层。
- 同一张摄影卡片不会重复朗读日期。
- 可访问树中同一卡片日期只出现一次；如果视觉重复不可避免，装饰重复项必须 `aria-hidden`。
- 日期语义元素或 `datetime` 保持机器可读。

### 摄影详情页移动端专项验收

问题：

- 详情页交互较丰富：顶部导航、作品内缩略图、灯箱、底部作品胶片条。
- 移动端最容易出现控件堆叠或误触。

建议：

- 用 iPhone 宽度检查导航、返回、灯箱关闭、左右滑动。
- 用 iPad 宽度检查缩略图 rail 和底部胶片条是否占位合理。
- 避免持续动画，保留轻量过渡。

验收：

- 手机上能顺利进入、浏览、退出作品。
- 灯箱不会遮挡关键控制。
- 底部作品胶片条可以横向滑动，并能明确识别当前作品。
- 灯箱打开后 body 不发生背景滚动。
- Esc 和关闭按钮均可退出灯箱，焦点不会丢失。
- 左右滑动不误触浏览器返回。
- 横竖屏切换后当前图片和胶片条状态保持合理。

### 内容发布 SOP 固化

问题：

- 当前发布依赖记忆 frontmatter。
- 摄影发布字段多，容易漏填。

建议：

- 使用 `docs/publishing-workflow.md` 作为发布标准。
- 每次发布前运行内容健康检查。
- 后续补充模板生成脚本。

验收：

- 新文章和新摄影作品都能按文档独立发布。
- 发布后无需回头猜字段含义。
- 发布前运行 `npm run check:content-health` 和 `npm run build`。
- 发布后抽查新内容 URL、列表入口、标签入口、RSS/sitemap 如项目启用、分享标题与描述。

## P1：近期改进

### UI 改动通用上线门禁

问题：

- 摄影页和普通内容页已经形成两套视觉系统，任何共享样式改动都可能造成跨页面断层。
- 只凭本地肉眼判断容易漏掉移动端、深色主题和可访问性问题。

建议：

- 每轮 UI 改动后执行截图或真实浏览器回归。
- UI 改动必须记录检查页面、视口、主题和是否触及共享样式。
- 视觉回归作为 P1 通用门禁，而不是后续观察项。

验收：

- 至少检查 `/articles/`、普通文章详情、任一标签页、`/photography/`、摄影详情页。
- 至少覆盖 `1440x900`、`768x1024`、`390x844`、`375x667`。
- 普通页浅色/深色主题可读，摄影页固定深色且无半浅半深断层。
- 无横向滚动、无明显布局跳动、移动端导航可用。

### 增加内容模板脚本

建议：

- `npm run new:post` 生成普通文章模板。
- `npm run new:photo` 生成摄影作品模板。
- 脚本只生成草稿，默认 `draft: true`。

验收：

- 不再需要从旧文章复制 frontmatter。
- 文件名、日期、基础字段自动生成。
- 生成文件默认 `draft: true`，避免误发布。
- slug 不覆盖已有文件。
- 空标题、非法标题或重复 slug 有明确提示。
- 生成后内容健康检查能通过，或给出可操作错误。

### 增强内容健康检查

建议：

- 检查普通文章和摄影文章的 required fields，并分开报错。
- 检查 `title` 是否为 `undefined`、`null`、空字符串或无意义占位。
- 检查摄影作品是否存在 `gallery[0]`。
- 检查本地图片路径是否存在。
- 检查摄影作品是否包含 `摄影` 标签。
- 检查过大的本地图片并给出提示。
- 检查重复 slug、未来日期、draft 是否被生产列表排除。

验收：

- 常见发布错误能在部署前发现。
- 报告输出能直接指导修复。
- 普通文章、摄影作品、draft 内容的错误边界清楚。

### 文章列表分页策略统一

问题：

- `/articles/` 当前全量展示非摄影文章。
- `/page/[page]` 已有静态分页，但入口心智不够统一。
- 线上真实体验确认当前数量仍可接受，但增长后页面会变长。

建议：

- 内容量继续增加后，将 `/articles/` 改为文章归档分页第一页，或增加明显分页入口。
- 保持静态分页，不做 infinite scroll。
- 当非摄影文章超过 30 篇时重新评估，不直接把首页分页心智混用为文章归档分页。

验收：

- 文章列表长度可控。
- 页码跳转明确，SEO 和静态构建保持简单。
- 分页启用前检查当前非摄影文章数量、分页入口、canonical、sitemap 和重复索引风险。

### 摄影归档增长策略

问题：

- `/photography/` 当前是单页年份归档。
- 作品继续增加后，页面图片数量会增加首屏和滚动负担。

建议：

- 内容量达到阈值后引入年份独立页，例如 `/photography/2024/`。
- `/photography/` 保留精选和年份入口。

验收：

- 摄影归档不会无限增长为重页面。
- 年份浏览路径清晰。
- 年份页和总摄影页不会制造重复入口混乱。

## P2：后续观察

### 标签页引导优化

建议：

- 保持静态权重标签，避免动态标签云造成资源消耗。
- 当内容量增长后，再考虑按主题分组标签。

验收：

- 标签页无明显性能负担。
- 标签数量增加后仍可扫描。

### 首页双主线整合

建议：

- 首页保持克制，只做入口和近期内容。
- 不新增解释设计的文案。
- 文章和摄影都要能一跳进入目标内容。

验收：

- 新访客 5 秒内知道可以读文章或看摄影。
- 首页不成为重复列表页。

### 作品胶片条提示增强

建议：

- 底部作品胶片条保留横向滑动，不回退到“较新 / 较早”两个按钮。
- 当前作品高亮可以更明确。
- 可滑动提示要轻，避免强遮罩和持续动画。

验收：

- 用户能理解底部区域可以继续横向浏览作品。
- 当前作品在胶片条中清晰可辨。

## 暂不做

- 不引入 CMS。
- 不做 Apple Notes / Shortcuts 发布集成。
- 不做 infinite scroll。
- 不恢复动态标签云。
- 不给摄影页做浅色主题。

## 文章页专项改进（Product + UI/UX）

### P1：文章页首屏语境与留白收口

问题：

- `/articles/` 当前阅读气质清爽，但首屏只展示 `随便聊聊`，对新访客的内容范围解释不足。
- 桌面端 hero 留白偏大，移动端标题视觉重量偏强，第一篇文章出现位置略靠下。
- 普通内容页 hero eyebrow 前的小图标属于装饰性信息，可以继续减少干扰。

建议：

- 在 `随便聊聊` 下增加一句克制说明：`工程实践、工具折腾和一些不太正式的记录。`
- 移除 `/articles/` hero eyebrow 前的小图标，只保留文字标签。
- 小幅压缩 `/articles/` hero 区域上下留白，优先优化移动端。
- 摘要文字略微加深，提高手机端可读性。

验收：

- `1440x900` 首屏能看到标题、说明文案和第一篇文章标题开头。
- `390x844` 首屏能看到标题、说明文案和第一篇文章标题开头。
- `375x667` 首屏至少能看到标题、说明文案和第一篇文章日期。
- 页面仍保持文本流，不变成重卡片布局。
- `document.documentElement.scrollWidth === document.documentElement.clientWidth`。

### P2：文章页内容发现增强

问题：

- 当前文章页主要依赖时间线浏览。
- 内容增长后，新访客不容易按主题快速进入。

建议：

- 在文章页增加 5-8 个高频标签入口，来源于非摄影文章 tags 统计。
- 标签入口保持低饱和、文本优先，不做动态标签云。
- 不引入 infinite scroll；非摄影文章超过 30 篇后再评估分页入口。

验收：

- 新访客可以通过标签进入主题内容。
- 标签入口不抢文章列表主视觉。
- 标签换行显示稳定，无横向滚动。
- 只统计 `!draft && type !== 'photography'` 的文章。
- 同 count 标签排序稳定，中文、英文、空格和特殊字符标签链接都能跳转。

### 执行团队与上线回归要求

执行角色：

- Product Manager：确认文章页目标、首屏用户任务和内容发现路径。
- UI/UX Designer：确认视觉收口、移动端密度、图标使用规则和文字层级。
- Sr. Front-end Engineer：负责实现 Astro 页面、collection 数据统计、响应式样式和主题兼容。
- QA Reviewer：负责桌面、手机、主题切换、长标题、无横向滚动、可访问性和线上回归验证。
- Code Reviewer：负责检查 scope、可维护性、无过度设计、无破坏摄影页固定深色主题。

上线顺序：

1. 本地实现后运行 `git diff --check`、`npm run build`、`npm run check:content-health`、`npm run check:security`。
2. 使用本地构建或线上预览检查 `/articles/` 桌面、iPad、手机和小屏手机视图。
3. 回归 `/`、任一普通文章详情页、任一 tag 页面、`/photography/`。
4. Code Review 确认只包含文章页专项相关改动，且共享样式有对应回归记录。
5. 提交并 push。
6. 部署完成后访问 `https://notes.miaowu.org/articles/` 做线上回归，确认首屏、移动端、主题切换、标签入口和普通文章详情页表现稳定。
