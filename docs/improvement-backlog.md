# 体验与发布改进 Backlog

**更新日期**：2026-04-19  
**排序原则**：优先减少真实使用摩擦，其次优化视觉一致性，最后再考虑增强功能。

## P0：优先处理

### 修复无标题摄影作品的页面标题

问题：

- 线上摄影详情页出现 `undefined | Joey's Notes` 浏览器标题。
- 这会影响专业感、分享标题和 SEO。

建议：

- 摄影详情页标题 fallback 使用日期或“摄影作品”。
- `BaseLayout` 的 `title` 入参不应收到 `undefined` 拼接结果。
- 对无 `title` 的摄影内容增加专门验收。

验收：

- 无标题摄影作品的浏览器标题不出现 `undefined`。
- 页面内也不强制显示“未命名作品”。

### 摄影列表页信息密度继续收口并移除重复日期

问题：

- 摄影列表页承担作品入口和归档浏览双重任务。
- 日期、相机、标题、按钮如果同时出现，容易削弱图片优先级。
- 线上可访问树显示归档卡片日期重复出现，说明 DOM 层也存在重复信息。

建议：

- 归档卡片默认只显示最必要 meta。
- 详细 EXIF 留给摄影详情页。
- featured 区继续避免明显卡片感，让图片本身做主视觉。
- 日期只保留一处，避免视觉和辅助技术重复读取。

验收：

- 首屏第一眼看到的是照片，而不是控件或说明。
- 手机端归档卡片高度稳定，没有明显断层。
- 同一张摄影卡片不会重复朗读日期。

### 摄影详情页移动端专项验收

问题：

- 详情页交互较丰富：顶部导航、作品内缩略图、灯箱、底部作品胶片条。
- 移动端最容易出现控件堆叠或误触。

建议：

- 用 iPhone 宽度检查菜单、返回、灯箱关闭、左右滑动。
- 用 iPad 宽度检查缩略图 rail 和底部胶片条是否占位合理。
- 避免持续动画，保留轻量过渡。

验收：

- 手机上能顺利进入、浏览、退出作品。
- 灯箱不会遮挡关键控制。
- 底部作品胶片条可以横向滑动，并能明确识别当前作品。

### 内容发布 SOP 固化

问题：

- 当前发布依赖记忆 frontmatter。
- 摄影发布字段多，容易漏填。

建议：

- 使用 `docs/publishing-workflow.md` 作为发布标准。
- 每次发布前运行内容健康检查。
- 后续补充模板生成脚本。

验收：

- 新文章和新摄影作品都能按文档独立发布。
- 发布后无需回头猜字段含义。

## P1：近期改进

### 增加内容模板脚本

建议：

- `npm run new:post` 生成普通文章模板。
- `npm run new:photo` 生成摄影作品模板。
- 脚本只生成草稿，默认 `draft: true`。

验收：

- 不再需要从旧文章复制 frontmatter。
- 文件名、日期、基础字段自动生成。

### 增强内容健康检查

建议：

- 检查摄影作品是否存在 `gallery[0]`。
- 检查本地图片路径是否存在。
- 检查摄影作品是否包含 `摄影` 标签。
- 检查过大的本地图片并给出提示。

验收：

- 常见发布错误能在部署前发现。
- 报告输出能直接指导修复。

### 文章列表分页策略统一

问题：

- `/articles/` 当前全量展示非摄影文章。
- `/page/[page]` 已有静态分页，但入口心智不够统一。
- 线上真实体验确认当前数量仍可接受，但增长后页面会变长。

建议：

- 内容量继续增加后，将 `/articles/` 改为分页第一页，或增加明显分页入口。
- 保持静态分页，不做 infinite scroll。

验收：

- 文章列表长度可控。
- 页码跳转明确，SEO 和静态构建保持简单。

### 摄影归档增长策略

问题：

- `/photography/` 当前是单页年份归档。
- 作品继续增加后，页面图片数量会增加首屏和滚动负担。

建议：

- 内容量达到阈值后引入年份独立页，例如 `/photography/2024/`。
- `/photography/` 保留精选和年份入口。

验收：

- 摄影归档不会无限增长为重页面。
- 年份浏览路径清晰。

## P2：后续观察

### 标签页引导优化

建议：

- 保持静态权重标签，避免动态标签云造成资源消耗。
- 当内容量增长后，再考虑按主题分组标签。

验收：

- 标签页无明显性能负担。
- 标签数量增加后仍可扫描。

### 首页双主线整合

建议：

- 首页保持克制，只做入口和近期内容。
- 不新增解释设计的文案。
- 文章和摄影都要能一跳进入目标内容。

验收：

- 新访客 5 秒内知道可以读文章或看摄影。
- 首页不成为重复列表页。

### 视觉回归检查

建议：

- 每轮 UI 改动后检查浅色普通页、深色摄影页和移动端导航。
- 优先人工截图比对，不急着引入重型视觉测试。

验收：

- 普通页和摄影页各自一致。
- 切换页面无明显底色断层、布局跳动或控件丢失。

### 作品胶片条提示增强

建议：

- 底部作品胶片条保留横向滑动，不回退到“较新 / 较早”两个按钮。
- 当前作品高亮可以更明确。
- 可滑动提示要轻，避免强遮罩和持续动画。

验收：

- 用户能理解底部区域可以继续横向浏览作品。
- 当前作品在胶片条中清晰可辨。

## 暂不做

- 不引入 CMS。
- 不做 Apple Notes / Shortcuts 发布集成。
- 不做 infinite scroll。
- 不恢复动态标签云。
- 不给摄影页做浅色主题。
